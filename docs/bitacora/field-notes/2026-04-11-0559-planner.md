## Contexto
El usuario reporto que el reporte de cobertura en PRs incluye assemblies de terceros (xunit, FluentValidation, Azure Functions Worker) ademas de los del proyecto.

## Descubrimientos
- `dotnet-coverage.settings.xml` tiene un Include con regex para Bitakora pero no filtra efectivamente los terceros en la recoleccion
- ReportGenerator no tenia ningun filtro de assemblies — reportaba todo lo que encontraba en el XML de cobertura
- El approach de excluir terceros uno a uno no escala; se descarto a favor de whitelist

## Decisiones
- Usar `assemblyfilters` de ReportGenerator con whitelist: `+Bitakora.ControlAsistencia.*;-*.Tests`
- El fix de recoleccion (dotnet-coverage) se deja como opcional — el problema visible se resuelve en la capa de reporte

## Descartado
- Excluir assemblies de terceros uno a uno en dotnet-coverage.settings.xml (no escala)
