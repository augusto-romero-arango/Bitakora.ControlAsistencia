## Contexto
Se necesitaba un workflow que ejecute smoke tests de todos los dominios en paralelo, y renombrar el actual para clarificar que es por dominio.

## Descubrimientos
- GitHub Actions no permite `workflow_call` dentro de un job con matrix strategy, así que el workflow global replica el step de `dotnet test` directamente.
- El `domain-scaffolder` crea workflows de deploy que referencian `smoke-tests.yml` — hay que actualizar esa referencia.

## Decisiones
- Registro de dominios en `.github/smoke-tests-dominios.json` (opción A: archivo JSON explícito) en lugar de descubrimiento dinámico en runtime o matrix hardcodeada en YAML.
- El scaffolder es responsable de appendear al JSON cuando crea un dominio nuevo.
- El workflow global usa job de discovery + matrix con `fromJSON`.

## Descartado
- Descubrimiento dinámico buscando `tests/*SmokeTests/*.csproj` en runtime — menos predecible y no obliga al scaffolder a mantener el registro.
- Matrix hardcodeada en el YAML del workflow — requiere parsear YAML para agregar entradas.
